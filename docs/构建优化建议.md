# 构建优化建议

基于 [Gradle 9.6 Performance Guide](https://docs.gradle.org/9.6.0/userguide/performance.html) 审查。

---

## 当前已启用的优化

| 优化项 | 配置值 | 说明 |
|--------|--------|------|
| 并行执行 | `org.gradle.parallel=true` | 多子项目任务并行 |
| 构建缓存 | `org.gradle.caching=true` | 缓存 task 输出，跳过重复执行 |
| 配置缓存 | `org.gradle.configuration-cache=true` | 缓存配置阶段结果 |
| JVM 堆内存 | `-Xmx12g` | 远超 Gradle 默认 512MB |

## 已采纳的修改

### 1. 移除 `isCrunchPngs = true`

- **文件**: `app/build.gradle.kts`
- **原因**: aapt2 已在打包时自动优化 PNG 资源，`isCrunchPngs` 触发额外冗余的 CPU 密集型压缩，拖慢 release 构建。

### 2. Debug 构建移除 ProGuard

- **文件**: `app/build.gradle.kts`
- **原因**: Debug 构建无需混淆，移除后可加速日常开发增量构建。
